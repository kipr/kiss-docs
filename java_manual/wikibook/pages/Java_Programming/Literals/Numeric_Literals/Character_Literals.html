Java Programming/Literals/Numeric Literals/Character Literals

Character literals are constant valued character expressions embedded in a Java program. Java characters are sixteen bit Unicode characters, ranging from 0 to 65535. Character literals are expressed in Java as a single quote, the character, and a closing single quote:

'a'
'7'
'$'
'π'

Character literals have the type char, an unsigned integer primitive type. Character literals may be safely promoted to larger integer types such as int and long. Character literals used where a short or byte is called for must be cast to short or byte since truncation may occur.

Mixed Mode Operations

Be careful when mixing character literals and integers in String concatenation operations. For example

int one = '1';
int zero = '0'; 
System.out.println("120? " + one + '2' + zero );
System.out.println("  3? " + (one + '2' + zero) );
System.out.println("102? " + 100 + '2' + 0 );
System.out.println("102? " + (100 + '2' + 0) );

will print the following values:

120? 49248
  3? 147
102? 10020
102? 150

The unexpected results arise because String concatenation will simply concatenate Strings and characters together, but integer values, including the result of integer expressions such as addition, are converted to strings. Since a character literal is really an integer type, expressions such as (one + '2' + zero) and (100 + '2' + 0) are evaluated as integer addition first, and the result is then converted to a String value for the String concatenation.

The first print statement concatenates "120? ", the String value of the integer value '1' which is 49; the character '2' and the integer value of '0' which is 48, yielding "120? " + "49" + "2" + "48", or "120? 49248".

The statements

char one = '1';
char zero = '0'; 
System.out.println("120? " + one + '2' + zero );

yield the desired

120? 120
