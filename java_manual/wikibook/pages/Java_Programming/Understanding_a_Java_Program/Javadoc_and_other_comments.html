Java Programming/Understanding a Java Program/Javadoc and other comments

Below is an enhancemnt to the Distance class which we presented in Understanding a Java Program. If you have not already read that module, please do so now to familiarize yourself with the purpose and structure of this sample class.

Let's review our Distance class as it was initially coded:

public class Distance
{
   private java.awt.Point point0, point1;

   public Distance(int x1, int y1, int x2, int y2)
   {
      point0 = new java.awt.Point(x1, y1);
      point1 = new java.awt.Point(x2, y2);
   }

   public void printDistance()
   {
      System.out.println("Distance between " + point0
                         + " and " + point1
                         + " is " + point0.distance(point1));
   }

   public static void main(String[] args)
   {
      Distance dist = new Distance(
            intValue(args[0]), intValue(args[1]),
            intValue(args[2]), intValue(args[3]));
      dist.printDistance();
   }

   private static int intValue(String data)
   {
      return Integer.parseInt(data);
   } 
}

One thing you may notice is the paucity of source code comments which describe the class and its implementation. This was intentional so that we could use prose in Understanding a Java Program to document the class.

While a small or cleanly defined Java program such as this one is often legible without comments, comments can aid in understanding more complex Java classes. Most notably, comments can help specify the application programming interface by describing what each method does, what values the methods return, and what the constraints of their inputs have.

The module Comments discusses Java comments in more detail. We will add sufficient comments so that someone looking at the source can understand how it is implemented and its the API and its constraints. This will make it easier to use the class elsewhere, to extend it by adding functionality, or to create new classes which extend it.

We will add comments immediately before each of the main declared elements: the class itself and its fields, constructor, and methods.

/**
 * Compute and print the distance between two points on a coordinate plane.
 */
public class Distance
{
   /** The first point */ 
   private java.awt.Point point0;
   /** The second point. This class computes the distance between point0 and point1 */ 
   private java.awt.Point point1;

   /**
    * Construct a Distance instance which will compute the distance between
    * the two points on the coordinate plane, (x0, y0) and (x1, y1)
    */
  public Distance(int x0, int y0, int x1, int y1)
  {
    point0 = new java.awt.Point(x0, y0);
    point1 = new java.awt.Point(x1, y1);
  }

  /**
   * Prints the two points that were passed to the constructor
   * and the distance between them.
   * The output is printed to the standard output, System.out.
   */
  public void printDistance()
  {
    System.out.println("Distance between " + point0 + " and " + point1 
                    + " is " + point0.distance(point1));
  }

  /**
   * The main entry point for executing this program from a command line.
   * This program requires four parameters which must be integer values
   * withing the range -2147483648 to 2147483647:
   * <pre>
   * java Distance x0 y0 x1 y1
   * </pre>
   * For example:
   * <pre>
   * java Distance 0 3 4 0
   * </pre>
   */
  public static void main(String[] args)
  {
    Distance dist = new Distance(
             intValue(args[0]), intValue(args[1]),
             intValue(args[2]), intValue(args[3]));
    dist.printDistance();
  }

  /**
   * Return the integer value as represented by the string data
   * For example, intValue("721") will return the integer 721 and
   * intValue("-9") will return the integer -9.
   * @param data A string that contains an integer value.
   */
  private static int intValue(String data)
  {
    return Integer.parseInt(data);
  }
}

You may notice that we split the decaration of the fields

   private java.awt.Point point0, point1;

into two separate declarations so that we could provide a separate comment for each field.
